# リグレッションテストルール

## 重要性

リグレッションテストはとても重要です。コードの変更後に既存の機能が正常に動作することを確認するために必須です。

## 必須実行ルール

バックエンド・フロントエンドをいじったあとは、**必ず**それぞれのディレクトリ配下で`make test`を実行して問題ないことを確認してください。

### 実行手順

1. **バックエンドの変更後**:
   ```bash
   cd crypto-trading-connector-be
   make test
   ```

2. **フロントエンドの変更後**:
   ```bash
   cd crypto-trading-connector-fe
   make test
   ```

3. **両方を変更した場合**:
   両方のディレクトリでテストを実行してください。

### テスト失敗時の対応

- テストが失敗した場合は、必ず修正してからコミットしてください
- テストの期待値が変更された場合は、テストコードも適切に更新してください
- 新機能追加時は、対応するテストも追加してください

### 例外

このルールに例外はありません。すべてのコード変更後にリグレッションテストを実行することが必須です。

## 自動化の推奨

可能であれば、CI/CDパイプラインでこれらのテストを自動実行することを推奨します。
